# 完整接口文档

## 1. 文档目的

这份文档记录三个模块之间共用的接口。内部算法和私有函数可以自由调整，但公开类、方法签名、数据格式和对外行为要对得上。如果确实需要改公开接口，先在组内确认，再同步更新这里。

项目依赖方向：

```text
config ← core ← app → network
             app → ui
ui → config、Piece
network → config
```

禁止 `core` 导入 `pygame`、`ui`、`network` 或 `app`。

## 2. `tetris.config` 常量接口

### 窗口与棋盘

| 名称 | 类型 | 当前值 | 含义 |
|---|---|---:|---|
| `WIDTH` | `int` | 1100 | 窗口宽度，像素 |
| `HEIGHT` | `int` | 760 | 窗口高度，像素 |
| `FPS` | `int` | 60 | 目标刷新率 |
| `COLS` | `int` | 10 | 棋盘列数 |
| `ROWS` | `int` | 20 | 棋盘行数 |
| `CELL` | `int` | 30 | 单机棋盘单格大小 |
| `BOARD_W` | `int` | 300 | `COLS * CELL` |
| `BOARD_H` | `int` | 600 | `ROWS * CELL` |

### 颜色

所有颜色都是 `(R, G, B)` 元组，每个分量范围为 0～255。

`BG`、`PANEL`、`PANEL_2`、`TEXT`、`MUTED`、`ACCENT`、`PINK`、`GREEN`、`RED` 分别用于背景、面板、按钮、主要文字、次要文字、强调和状态颜色。

`COLORS: dict[str, tuple[int, int, int]]` 必须包含七个键：`I O T S Z J L`。

### 网络

| 名称 | 类型 | 当前值 | 含义 |
|---|---|---:|---|
| `DISCOVERY_PORT` | `int` | 45678 | UDP 房间发现端口 |
| `GAME_PORT` | `int` | 45679 | TCP 游戏端口 |
| `MATCH_SECONDS` | `int` | 60 | 联机比赛时长 |

## 3. `tetris.core` 接口

### 3.1 方块类型

`PieceKind` 在当前实现中使用字符串表示：

```python
"I" | "O" | "T" | "S" | "Z" | "J" | "L"
```

棋盘锁定格使用上述字符串，空格在本地棋盘中使用 `None`，在网络快照中使用 `""`。

### 3.2 `SHAPES`

```python
SHAPES: dict[str, tuple[str, str, str, str]]
```

每个类型对应一个 4×4 字符模板。`.` 表示空白，类型字母表示实体格。每种形状必须恰好有四个实体格。

### 3.3 `rotate_cells`

```python
rotate_cells(
    cells: list[tuple[int, int]],
    turns: int
) -> list[tuple[int, int]]
```

- `cells`：4×4 模板中的局部坐标列表。
- `turns`：顺时针旋转次数；通过 `% 4` 归一化。
- 返回值：新的坐标列表，不修改输入列表。
- 单次旋转规则：`(x, y) -> (3-y, x)`。

### 3.4 `Piece`

```python
Piece(kind: str, x: int = 3, y: int = -1, rotation: int = 0)
```

公开字段：

| 字段 | 类型 | 含义 |
|---|---|---|
| `kind` | `str` | 七种方块类型之一 |
| `x` | `int` | 4×4 模板左上角对应的棋盘列 |
| `y` | `int` | 4×4 模板左上角对应的棋盘行 |
| `rotation` | `int` | 0～3，对应四个方向 |

#### `Piece.cells`

```python
cells(
    x: int | None = None,
    y: int | None = None,
    rotation: int | None = None
) -> list[tuple[int, int]]
```

- 未提供参数时使用对象当前字段。
- 参数只用于计算候选状态，不应修改对象。
- 返回四个实体格的绝对棋盘坐标。
- 坐标允许 `y < 0`，代表方块尚未完全进入棋盘。

### 3.5 `Bag`

```python
Bag(seed: object | None = None)
```

公开字段：

- `random: random.Random`
- `items: list[str]`

#### `Bag.pop`

```python
pop() -> str
```

- 返回一个方块类型。
- 袋子为空时创建七种类型并打乱。
- 任意一轮连续七次取出应恰好包含七种类型。
- 相同 seed 的对象应生成相同序列。

### 3.6 `ScoreSystem`（组长负责）

独立计分器，不接触棋盘和方块：

```python
ScoreSystem()
```

类常量：

```python
CLEAR_POINTS = {1: 100, 2: 300, 3: 500, 4: 800}
```

公开字段：

| 字段 | 类型 | 初始值 | 含义 |
|---|---|---:|---|
| `score` | `int` | 0 | 当前总分 |
| `lines` | `int` | 0 | 累计消行数 |
| `level` | `int` | 1 | 当前等级 |
| `combo` | `int` | -1 | 连击索引 |

#### `award_soft_drop`

```python
award_soft_drop(distance: int = 1) -> None
```

按 `max(0, distance)` 增加分数，即每格 1 分。

#### `award_hard_drop`

```python
award_hard_drop(distance: int) -> None
```

按 `max(0, distance) * 2` 增加分数，即每格 2 分。

#### `apply_line_clear`

```python
apply_line_clear(count: int) -> int
```

- `count <= 0`：将 `combo` 重置为 `-1`，返回 0。
- `count` 为 1～4：累加消行，更新 `level = lines // 10 + 1`。
- 连续消行时 `combo += 1`。
- 本次分数为 `CLEAR_POINTS[count] * level + max(0, combo) * 50`。
- 返回本次获得的消行分。

`ScoreSystem` 的全部实现与测试由组长负责。其他成员只能调用上述接口，不得在 `TetrisGame` 中重复实现分值公式。

### 3.7 `TetrisGame`（成员一负责）

```python
TetrisGame(seed: object | None = None)
```

#### 类常量

```python
SCORE = ScoreSystem.CLEAR_POINTS
KICKS = [(0,0), (-1,0), (1,0), (-2,0), (2,0), (0,-1)]
```

#### 公开状态字段

| 字段 | 类型 | 初始值 | 含义 |
|---|---|---|---|
| `board` | `list[list[str | None]]` | 20×10 空棋盘 | 已锁定格 |
| `queue` | `list[str]` | 5 个类型 | 后续方块队列 |
| `current` | `Piece` | 队首方块 | 当前活动块 |
| `hold` | `str | None` | `None` | 暂存方块 |
| `can_hold` | `bool` | `True` | 当前活动块是否可暂存 |
| `scoring` | `ScoreSystem` | 新计分器 | 组长提供的计分对象 |
| `score` | 只读 `int` 属性 | 0 | 转发 `scoring.score` |
| `lines` | 只读 `int` 属性 | 0 | 转发 `scoring.lines` |
| `level` | 只读 `int` 属性 | 1 | 转发 `scoring.level` |
| `combo` | 只读 `int` 属性 | -1 | 转发 `scoring.combo` |
| `game_over` | `bool` | `False` | 是否结束 |
| `paused` | `bool` | `False` | 是否暂停 |
| `fall_timer` | `float` | 0.0 | 自动下落累计秒数 |
| `lock_timer` | `float` | 0.0 | 落地锁定累计秒数 |

#### `fall_interval`

只读属性：

```python
game.fall_interval -> float
```

公式：`max(0.07, 0.8 * 0.86 ** (level - 1))`，单位为秒/格。

#### `valid`

```python
valid(
    piece: Piece | None = None,
    x: int | None = None,
    y: int | None = None,
    rotation: int | None = None
) -> bool
```

- 检查候选方块是否超出左右/底部边界或与锁定格重叠。
- `y < 0` 合法；`y >= ROWS` 非法。
- 不修改任何游戏状态。

#### `move`

```python
move(dx: int, dy: int) -> bool
```

- 成功则修改 `current.x/y` 并返回 `True`。
- 失败不修改位置并返回 `False`。
- 游戏结束或暂停时返回 `False`。
- 水平移动成功后重置锁定计时。

#### `rotate`

```python
rotate(direction: int = 1) -> bool
```

- `1` 为顺时针，`-1` 为逆时针。
- O 方块直接返回 `False`。
- 按 `KICKS` 顺序尝试旋转位置。
- 成功时更新位置和 `rotation`，重置锁定计时。
- 全部候选失败、暂停或游戏结束时返回 `False`。

#### `soft_drop`

```python
soft_drop() -> bool
```

- 尝试向下移动一格。
- 成功调用 `scoring.award_soft_drop()` 并返回 `True`。
- 失败返回 `False`，不加分。

#### `ghost_y`

```python
ghost_y() -> int
```

返回当前方块能够到达的最低模板 y，不修改当前位置。

#### `hard_drop`

```python
hard_drop() -> None
```

- 移动到 `ghost_y()`。
- 调用 `scoring.award_hard_drop(distance)` 计算下落分。
- 立即调用锁定，不使用 0.5 秒延迟。
- 暂停或结束时无操作。

#### `swap_hold`

```python
swap_hold() -> None
```

- 首次暂存：当前类型进入 `hold`，从队列生成下一块。
- 非首次：交换 `current.kind` 与 `hold`，交换出来的块以默认位置和旋转生成。
- 每个活动块最多调用成功一次。
- 交换后的方块位置非法时设置 `game_over=True`。

#### `spawn`

```python
spawn() -> None
```

- 从 `queue[0]` 创建默认位置的新 `Piece`。
- 从 `Bag` 补充队列，使其仍有五个元素。
- 设置 `can_hold=True`、`lock_timer=0`。
- 出生位置非法时设置 `game_over=True`。

#### `lock`

```python
lock() -> None
```

- 将活动块写入 `board`。
- 若任意实体格仍在顶部之外，设置游戏结束并返回。
- 删除所有完整行，在顶部补等量空行。
- 将消除行数传给 `scoring.apply_line_clear(n)`；计分器统一更新分数、累计行数、等级和连击。
- 最后生成下一块。

#### `update`

```python
update(dt: float) -> None
```

- `dt` 是上一帧到当前帧经过的秒数。
- 暂停或游戏结束时不更新。
- 按 `fall_interval` 自动向下移动。
- 落地持续约 0.5 秒后锁定。
- 应用层应把异常大的 `dt` 限制在合理范围，当前上限为 0.05 秒。

#### `snapshot`

```python
snapshot() -> dict
```

返回结构：

```python
{
    "board": list[list[str]],  # 20×10，空格为 ""，已合并活动块
    "score": int,
    "lines": int,
    "level": int,
    "game_over": bool,
}
```

快照必须可由 `json.dumps()` 序列化，且不得修改本地 `board`。

## 4. `tetris.ui` 接口

所有绘制函数直接修改传入的 `pygame.Surface`。

### `font`

```python
font(size: int, bold: bool = False) -> pygame.font.Font
```

- Windows 使用微软雅黑，macOS 使用华文黑体。
- 相同参数应返回缓存字体对象。

### `text`

```python
text(surface, value, size, color, pos, center=False, bold=False) -> pygame.Rect
```

- `value` 自动调用 `str()`。
- `center=False` 时 `pos` 是左上角；为真时是中心。
- 返回文字最终占用矩形。

### `panel`

```python
panel(surface, rect, radius=16, color=PANEL) -> None
```

绘制实心圆角矩形内容区。

### `Button`

```python
Button(rect, label: str, accent=ACCENT)
```

公开字段：

- `rect: pygame.Rect`
- `label: str`
- `accent: tuple[int, int, int]`

方法：

```python
draw(surface, mouse: tuple[int, int]) -> None
clicked(event: pygame.event.Event) -> bool
```

`clicked()` 只在鼠标左键按下且坐标位于按钮内部时返回真。

### `draw_block`

```python
draw_block(surface, rect, color, ghost=False) -> None
```

普通块为实心圆角矩形；幽灵块为 2 像素轮廓。方格应略小于传入矩形，以保留格间间隔。

### `draw_board`

```python
draw_board(
    surface,
    board,
    origin: tuple[int, int],
    cell: int,
    current: list[tuple[int, int, str]] | None = None,
    ghost: list[tuple[int, int, str]] | None = None,
    label: str | None = None,
) -> None
```

- `board` 必须至少为 20×10。
- 屏幕位置换算：`pixel_x=origin_x+x*cell`，`pixel_y=origin_y+y*cell`。
- 绘制顺序：网格/锁定格 → 幽灵格 → 当前格 → 标签。
- `current` 和 `ghost` 中 `y<0` 的格子不绘制。
- 可通过不同 `cell` 复用为本地和对手棋盘。

### `draw_preview`

```python
draw_preview(surface, kind: str | None, rect, title: str) -> None
```

- 始终绘制面板和标题。
- `kind is None` 时不绘制形状。
- 方块应按实体格包围盒在面板内居中，而不是按 4×4 模板居中。

## 5. `tetris.network` 接口

### `local_ip`

```python
local_ip() -> str
```

返回本机局域网 IPv4 字符串；无法判断时返回 `127.0.0.1`。

### `LanSession`

```python
LanSession()
```

公开状态：

| 字段 | 类型 | 含义 |
|---|---|---|
| `is_host` | `bool` | 当前是否为主机 |
| `connected` | `bool` | TCP 是否连接 |
| `status` | `str` | 供 UI 显示的中文状态 |
| `peer_name` | `str` | 对手显示名 |
| `inbox` | `queue.Queue` | 网络线程送往主线程的消息 |
| `outbox` | `queue.Queue` | 主线程送往网络线程的消息 |

#### `host`

```python
host(name: str = "主机") -> None
```

关闭旧会话，启动 UDP 房间应答和 TCP 监听。方法本身不能阻塞 Pygame 主线程。

#### `connect`

```python
connect(address: str, name: str = "玩家") -> None
```

在后台连接 `address:GAME_PORT`；成功后发送 `hello`。

#### `discover`

```python
LanSession.discover(timeout: float = 1.0) -> list[tuple[str, str]]
```

返回 `(IP, 房间名)` 列表，同一 IP 只保留一次。该调用最多阻塞约 `timeout` 秒，所以应用层必须放在后台线程调用。

#### `send`

```python
send(message: dict) -> None
```

非阻塞地放入发送队列。未连接时忽略。队列满时允许丢弃较旧状态快照，避免延迟持续累积。

#### `poll`

```python
poll() -> list[dict]
```

立即返回当前全部待处理消息；没有消息时返回空列表，不阻塞。

#### `close`

```python
close() -> None
```

通知线程停止，关闭 TCP 连接和监听套接字；可以重复调用。

以下划线开头的 `_thread`、`_advertise`、`_accept`、`_connect_worker`、`_activate`、`_send_now`、`_io_loop` 是内部接口，其他模块不得直接调用。这些线程方法接收当前会话的停止令牌，旧会话关闭后不得激活或覆盖新会话状态。

## 6. 网络协议

### 6.1 UDP 发现协议

编码：UTF-8 JSON；单个 UDP 数据报就是一条完整消息。

客户端广播：

```json
{"type":"discover"}
```

主机响应：

```json
{"type":"room","name":"DESKTOP-ROOM"}
```

### 6.2 TCP 帧格式

使用 UTF-8 JSON Lines：

```text
JSON对象 + \n
```

接收方必须缓存字节流并按换行拆包，因为一次 `recv()` 可能收到半条、一条或多条消息。

### 6.3 TCP 消息

#### 昵称交换

```json
{"type":"hello","name":"player-name"}
```

双方连接成功后均可发送。收到后更新 `peer_name`。

#### 开始比赛

```json
{"type":"start","delay":1.5}
```

- 仅主机发起。
- `delay` 表示接收消息后等待的秒数，当前为 1.5。
- 双方分别用自己的本地时间建立倒计时，因此不依赖系统时钟同步。

#### 游戏状态

```json
{
  "type":"state",
  "data":{
    "board":[["", "I"]],
    "score":1200,
    "lines":8,
    "level":1,
    "game_over":false
  }
}
```

`data` 必须符合 `TetrisGame.snapshot()`。当前发送周期约为 0.08 秒。

#### 最终分数

```json
{"type":"final","score":3200}
```

60 秒倒计时结束后双方各发送一次。应用应优先使用对手的 `final` 分数判定结果；等待两秒仍未收到时，才使用最近的 `state.score`，避免网络延迟导致双方结果不一致。

#### 本地产生的断线通知

```json
{"type":"disconnect","message":"对方已断开"}
```

此消息放入本地 `inbox`，不要求发送到已经失效的连接。

#### 本地产生的错误通知

```json
{"type":"error","message":"错误详情"}
```

用于 UDP 监听等后台错误；应用层可显示但不应崩溃。

## 7. `tetris.app.TetrisApp` 接口

### 构造

```python
TetrisApp()
```

重要字段：

| 字段 | 类型 | 含义 |
|---|---|---|
| `screen` | `pygame.Surface` | 主窗口画布 |
| `clock` | `pygame.time.Clock` | 帧率时钟 |
| `running` | `bool` | 主循环开关 |
| `scene` | `str` | `menu/single/network/battle` |
| `game` | `TetrisGame | None` | 当前本地游戏 |
| `session` | `LanSession` | 当前网络会话 |
| `remote` | `dict | None` | 最新对手快照 |
| `match_start` | `float | None` | 本机单调时钟中的比赛开始点 |
| `match_finished` | `bool` | 比赛是否结束 |
| `result` | `str` | 胜负提示 |
| `address` | `str` | 联机大厅输入的 IP |
| `discovery_status` | `str` | 搜索结果提示 |
| `last_sync` | `float` | 上次发送快照时间 |

### 生命周期方法

```python
run() -> None
handle_global(events: list) -> None
```

`run()` 每帧限制 `dt <= 0.05`，根据 `scene` 调用对应场景，最后执行 `pygame.display.flip()`；退出时必须关闭网络并调用 `pygame.quit()`。

### 场景方法

```python
menu(events) -> None
start_single() -> None
network_lobby(events) -> None
start_battle(start_at: float) -> None
leave_network() -> None
play(events, dt: float, battle: bool) -> None
```

- `start_single()` 创建新 `TetrisGame` 并进入 `single`。
- `start_battle()` 接收由 `time.monotonic()` 计算的本地开始点，创建本地游戏和 20×10 空远程快照并进入 `battle`。
- `leave_network()` 关闭旧会话、创建干净会话并返回菜单。
- `play()` 是单机与对战共用的更新入口。

### 输入方法

```python
input_game(events, battle: bool) -> None
```

| 按键 | 调用 |
|---|---|
| Left | `game.move(-1, 0)` |
| Right | `game.move(1, 0)` |
| Down | `game.soft_drop()` |
| X / Up | `game.rotate(1)` |
| Z | `game.rotate(-1)` |
| Space | `game.hard_drop()` |
| C | `game.swap_hold()` |
| P | 单机切换 `game.paused` |
| Esc | 返回菜单或离开联机 |

### 网络更新

```python
network_update(now: float, remaining: float) -> None
```

- `state` 更新 `remote`。
- `disconnect` 结束比赛并显示提示。
- 每隔至少 0.08 秒发送一次本地快照。
- `remaining <= 0` 时发送本地 `final`，收到对手最终分数后设置 `胜利！/惜败/平局`。

### 绘制辅助

```python
background() -> None
draw_game(battle, countdown, remaining) -> None
current_cells() -> list[tuple[int, int, str]]
ghost_cells() -> list[tuple[int, int, str]]
draw_single() -> None
draw_battle(countdown, remaining) -> None
overlay(title, subtitle) -> None
```

`current_cells()` 和 `ghost_cells()` 是规则对象到 UI `draw_board()` 数据格式的适配层。

## 8. 模块间使用示例

### 核心与 UI

game = TetrisGame(seed=1)

## 9. 哪些可以改，哪些不要动

### 允许

- 将内部列表推导式改成循环。
- 更换类的私有实现。
- 增加类型别名、数据类和私有辅助方法。
- 将 `TetrisApp` 的场景拆成独立类。
- 增加测试和错误日志。
- 改进线程关闭方式，只要公开行为不变。

### 禁止

- 在核心规则中直接读取键盘或绘制界面。
- 让 UI 直接修改棋盘、分数或等级。
- 使用 `pickle` 等不安全格式传输网络数据。
- 在 Pygame 主线程执行阻塞式 `accept()`、`connect()` 或长时间 `recv()`。
- 未更新接口文档就改变快照键名、网络消息类型或公开方法签名。
